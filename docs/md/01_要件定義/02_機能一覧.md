# Groove Match - 機能一覧

> 機能の優先順位: **1. マッチング（仲間探し）→ 2. コピバン結成（セットリスト管理）→ 3. ライブ出演（イベントボード）** がメインフロー。
> 練習トラッカー・フレンド・ランキングは「挫折防止のサポート機能」です。

アプリ全体像については [01_アプリ概要.md](./01_アプリ概要.md) を参照してください。

---

## 0. ユーザーストーリー（主要シナリオ）

各機能が「誰のために・何を解決するか」を定義します。

### メインフロー（仲間探し → コピバン結成 → ライブ出演）

| # | ユーザーストーリー | 関連機能 |
|---|------------------|---------|
| US-01 | **初心者ギタリストとして**、やりたい曲名で検索して同じ曲をやりたい仲間を見つけたい。**SNSや掲示板で探す手間をなくすために。** | マッチング、Spotify連携 |
| US-02 | **マッチングしたユーザーとして**、相手とチャットして練習日程を調整したい。**スムーズにバンド活動を始めるために。** | チャット、練習カレンダー |
| US-03 | **バンドリーダーとして**、コピバンを作成しメンバーを招待してセットリストを管理したい。**バンド運営を一つのアプリで完結するために。** | バンド管理、セットリスト |
| US-04 | **コピバンメンバーとして**、初心者歓迎のライブイベントを探してバンドで応募したい。**ライブデビューの機会を見つけるために。** | イベントボード |

### サポートフロー（挫折防止）

| # | ユーザーストーリー | 関連機能 |
|---|------------------|---------|
| US-05 | **練習中のユーザーとして**、日々の練習時間を記録してストリークを維持したい。**モチベーションを保って練習を続けるために。** | 練習トラッカー、バッジ |
| US-06 | **フレンドがいるユーザーとして**、フレンドの練習状況やランキングを見たい。**互いに刺激を受けて切磋琢磨するために。** | フレンド、ランキング |
| US-07 | **コミュニティメンバーとして**、練習ログやライブの感想を投稿したい。**バンド活動の記録を共有するために。** | コミュニティ |

### 管理・課金

| # | ユーザーストーリー | 関連機能 |
|---|------------------|---------|
| US-08 | **Freeユーザーとして**、Proプランに登録して無制限にマッチング・チャットを使いたい。**本気でバンド活動を始めるために。** | サブスクリプション |
| US-09 | **管理者として**、ユーザー通報を確認して不正ユーザーに対処したい。**健全なコミュニティを維持するために。** | 管理ダッシュボード、通報管理 |
| US-10 | **Proユーザーとして**、自主企画のライブイベントを登録して出演バンドを募集したい。**コピバンライブの場を自分で作るために。** | イベント作成 |

## 1. ユーザー登録・認証

- **ソーシャルログイン**: Google / X (Twitter) / Instagram アカウントでワンクリック登録・ログイン
- メールアドレス + パスワードによる認証（Amazon Cognito = AWSの認証サービス）
- マルチステップ登録フォーム（基本情報 → 音楽プロフィール → コピバン設定 → 地域・スケジュール → SNS連携）
- **初回登録時にProプラン選択が必須**（Web: Stripeでカード登録 / iOS: Apple IDの支払い方法 / Android: Google Playの支払い方法）
- 招待コード入力欄（任意）: 入力すると**トライアル期間が2週間→1ヶ月に延長**
- パスワード要件: 8文字以上、英字+数字必須
- JWT（JSON Web Token = サーバーがユーザーを識別するための暗号化された文字列）ベースのセッション管理
- クロスタブ同期対応

## 1.5. オンボーディング（初回チュートリアルミッション）

初回ログイン時に**ステップ形式のミッション**を表示し、アプリの使い方を自然に理解してもらいます。ゲーミフィケーション要素を取り入れ、全ミッション完了でバッジを獲得できます。

### ミッション一覧

| # | ミッション | 内容 | 完了条件 | 報酬 |
|---|-----------|------|---------|------|
| 1 | **プロフィールを完成させよう** | 楽器・ジャンル・やりたい曲・活動地域を入力 | プロフィール完成度80%以上 + メール認証完了 | 信頼レベル Lv.0→Lv.1 に昇格 |
| 2 | **最初のマッチングを探そう** | マッチング画面を開いて候補を閲覧 | マッチング画面に1回アクセス | ─ |
| 3 | **気になる人にいいねしよう** | マッチング候補に「いいね」を送信 | いいね1回送信 | ─ |
| 4 | **コピバンを作ろう or 参加しよう** | バンド作成画面で新規バンドを作成、またはバンドに参加 | バンド1つに所属 | ─ |
| 5 | **練習を記録してストリークを始めよう** | 練習トラッカーで練習時間を1回記録 | 練習ログ1件記録 | 🏅「Groove Matchデビュー」バッジ獲得 |

### 表示仕様

| 項目 | 仕様 |
|------|------|
| 表示タイミング | **初回ログイン直後**にダッシュボード上部に表示 |
| 進捗表示 | プログレスバー（5段階）で達成度を可視化。「2/5 完了」のように表示 |
| スキップ | **可能**。「後で」ボタンでミッションパネルを非表示にできる |
| 再表示 | 未完了ミッションがある場合、ダッシュボードに小さなバナーで「ミッション未完了」を表示 |
| 完了時 | 全5ミッション完了で**祝福アニメーション** + バッジ獲得通知 |
| 対象 | **新規登録ユーザーのみ**（既存ユーザーには表示しない） |

### ミッションの狙い

| ミッション | 狙い |
|-----------|------|
| 1. プロフィール完成 | **信頼レベルLv.1昇格**のトリガー。プロフィール未入力だとマッチング精度が低くなることを防ぐ。ミッション1〜3は順序依存（1未完了で3は実行不可） |
| 2. マッチング閲覧 | コアバリューである「仲間探し」を最初に体験させる |
| 3. いいね送信 | **アクティブなアクション**を早期に促し、マッチング成立の確率を上げる |
| 4. バンド作成/参加 | メインフローの「コピバン結成」への導線 |
| 5. 練習記録 | ストリーク開始で**継続利用のきっかけ**を作る。DAU/MAU比の向上に貢献 |

## 2. プロフィール管理

- 基本情報: 名前、ニックネーム、アバター、自己紹介
- **年代表示**: 生年月日から「20代前半」「30代後半」のように表示（生年月日は非公開）
- 音楽情報: 楽器（複数可）、スキルレベル、ジャンル（複数可）
- コピバン情報: コピーしたいアーティスト、やりたい曲、弾ける曲
  - 曲の検索・登録はSpotify API連携（詳細は [外部連携要件](./11_外部連携要件.md) を参照）
- 活動情報: 都道府県、市区町村、練習スケジュール、影響を受けたアーティスト
  - 都道府県・市区町村はRESAS API連携で選択（詳細は [外部連携要件](./11_外部連携要件.md) を参照）
- 表示/編集モード切り替え

## 3. コピバンマッチング

複数の要素（地域・楽器・ジャンル・スキル・スケジュール・曲マッチング）を総合的に判定し、相性の良いユーザーを自動で提案します。

> スコアリングの計算式・各要素の詳細ロジック・計算例は [マッチングアルゴリズム](../02_基本設計/01_マッチングアルゴリズム.md) を参照。

**フィルター:** やりたい曲、アーティスト、ジャンル、楽器、地域、スキルレベル

### マッチング制限（Free/Pro）

| 項目 | Free | Pro |
|------|------|-----|
| 1日あたりのいいね数 | 3回/日 | 無制限 |
| スコア詳細表示 | 非表示 | 表示 |
| フィルター要素 | 地域・楽器のみ | 全6要素 |
| AIマッチング（Embeddingブースト） | 非対応 | 対応（ローンチから） |
| マッチング履歴保持期間 | 30日 | 無制限 |

> **Freeユーザーのマッチング後導線**: Freeユーザーもマッチング成立は可能。チャット画面に「チャットを始めるにはProプランが必要です」バナーを表示し、Pro転換を促す。

## 4. チャット（リアルタイム通信）

マッチング後のコミュニケーション機能です。リアルタイムチャットをローンチ時から提供します。

- **1対1チャット**: マッチング成立後のダイレクトチャット（Pro限定）
- **バンドグループチャット**: コピバンメンバー間のグループチャット
- **練習日程調整**: アプリ内で練習日時・場所の候補を提示し、メンバーの参加可否を確認できる
- **練習場所共有**: スタジオ名・住所・地図リンクの共有機能
- **ハイブリッド方式**: 初回はアプリ内チャット → 信頼構築後にLINE等へ自然に移行

> チャットの安全機能（禁止ワードフィルタ・URL警告・レート制限・メッセージ通報）およびコミュニケーション設計の詳細は [安全対策・リスク管理](./04_安全対策・リスク管理.md) を参照。

## 5. コピバン管理

- コピバン作成（名前、説明、コピー対象アーティスト/曲、ジャンル、地域）
- **セットリスト管理**: コピーする曲の一覧・順序管理（Spotify APIで曲情報を取得）
- 楽器スロット制（空きポジション表示）: コピバンに「何が足りないか」が一目瞭然
- メンバー管理（リーダー/メンバー権限）
- 募集中フラグ
- コピバン詳細・編集画面
- **練習カレンダー**（Pro限定）: バンド単位の月表示カレンダーで練習日程を管理
  - メンバーが練習日程を提案（日時・スタジオ名・住所・メモ）
  - メンバーが OK / NG で回答
  - バンドリーダーが「確定」ボタンで確定
  - カレンダー上に色分け表示（提案中=黄、確定=緑、完了=灰、キャンセル=赤）
  - 前日にリマインド通知

## 8. 挫折防止サポート機能（Duolingo型ゲーミフィケーション）

> メインゴール「コピバンを組んでライブに出る」に到達するまでの挫折を防ぐサポート機能群です。

- **練習トラッカー** - 曲ごとの練習時間記録、ストリーク（連続練習日数）、週間目標
- **フレンド機能** - **相互承認型**（フォロー/フォロワー形式ではなく、双方が承認して初めてフレンドになる）
  - 練習状況の相互共有、応援機能
  - **練習ランキングは相互フレンド間で表示**
- **バッジ＆マイルストーン** - 12種類のバッジ（ストリーク、累計時間、初ライブ等）
- **通知システム** - 練習リマインド、ストリーク危機、フレンド活動、ランキング変動
- **ソーシャルログイン** - Google / X / Instagram でのアカウント作成・ログイン
- **SNS連携** - X / Instagram / LINE / YouTube アカウント連携（プロフィール表示用）

## 9. サブスクリプション・課金モデル

### プラン比較

| 機能 | Free (¥0) | Pro (¥500/月) |
|------|-----------|---------------|
| プロフィール作成 | ○ | ○ |
| マッチング検索 | 1日3回 | **無制限** |
| コミュニティ閲覧 | ○ | ○ |
| イベント閲覧 | ○ | ○ |
| バンド作成 | 1つまで | **無制限** |
| チャット | - | **○** |
| イベント作成 | - | **○（「公式」バッジなし）** |
| アナリティクス | - | **○** |
| フレンド | 5人まで | **無制限** |
| ランキング | フレンドのみ | **全体ランキング** |
| 通知カスタマイズ | 基本のみ | **全カスタマイズ** |
| SNS連携 | 1つ | **無制限** |

### 課金フロー

1. **初回登録時**: Proプランが自動選択され、**クレジットカード登録が必須**
2. **トライアル期間**: 2週間無料（招待コード入力で**1ヶ月に延長**）
3. **トライアル終了後**: 解約しなければ**自動で¥500/月の課金が開始**
4. **解約**: いつでも可能。解約するとFreeプランにダウングレード
5. **自動更新**: Freeプランにダウングレードしない限り毎月自動更新

### 法的注意事項（必須表示）

- 登録時に「トライアル後は自動でPro ¥500/月に移行する」旨を**明確に表示**
- **解約方法を分かりやすく案内**する
- **トライアル終了3日前にメール通知**を送信

### 招待コード機能

既存ユーザーが新規ユーザーを招待できる仕組みです。

| ステップ | 内容 |
|---------|------|
| 1 | 既存ユーザーがアプリ内で**招待コードを生成** |
| 2 | 新規ユーザーが登録時に招待コードを入力 |
| 3 | 新規ユーザー: トライアル期間が**2週間 → 30日に延長** |
| 4 | 紹介者: **Proプラン1週間延長**の特典 |

### 決済戦略（プラットフォーム別）

| 決済経路 | 手数料 | ¥500の手取り | 実装 |
|---------|--------|-------------|------|
| Web（Stripe） | 3.6% + ¥30/件 | ¥452 | Stripe Checkout |
| iOS（App Store） | 15%※ | ¥425 | StoreKit 2 |
| Android（Google Play） | 15%※ | ¥425 | Google Play Billing |

> ※ Apple Small Business Program / Google Play サブスク初年度料率を適用（年間売上$1M以下）
> ※ Stripeは決済額の3.6%に加え、1件あたり¥30の固定手数料が発生

**併用戦略**: Web経由はStripe決済（手取り最大）、iOS/AndroidはApp内課金（UXシームレス）を併用します。
加重平均手取りは **¥436/人** となります。

詳細なコスト計算は [01_コスト計算.md](../04_コスト・運用/01_コスト計算.md) を参照してください。

## 10. 管理ダッシュボード（管理者専用）

**アクセス制限**: admin権限を持つユーザーのみアクセス可能。一般ユーザーには表示されない。

### 10.1 概要ダッシュボード（トップ画面）

ログイン直後に表示される全体サマリー。主要KPIを一目で把握できる。

| カード | 表示内容 | 更新頻度 |
|--------|---------|---------|
| **総ユーザー数** | 登録ユーザーの累計数。前月比の増減（+120 ↑12%）も表示 | リアルタイム |
| **MAU（月間アクティブ）** | 当月のMAU。DAU/MAU比もゲージで表示（目標30%） | 日次 |
| **DAU（日間アクティブ）** | 本日のDAU。過去7日間の推移を折れ線グラフで表示 | リアルタイム |
| **Pro会員数** | 有料会員数と転換率（目標10〜15%）。前月比も表示 | リアルタイム |
| **月間売上** | 当月の売上見込み（¥）。前月実績との比較 | 日次 |
| **通報件数（未対応）** | 未処理の通報件数。赤バッジで緊急度を強調 | リアルタイム |

### 10.2 ユーザーアクティビティ分析

| 画面 | 表示内容 |
|------|---------|
| **登録推移グラフ** | 日別/週別/月別の新規登録数。折れ線グラフ。登録ソース（メール/Google/X/Instagram）別の内訳も表示 |
| **登録完了率** | 登録フォーム開始→完了の転換率。ステップ別の離脱率（Step1: 95%, Step2: 88%, ... 完了: 72%）をファネルで可視化 |
| **オンボーディング完了率** | 初回チュートリアルミッションの各ステップ完了率。どのミッションで離脱しているかを把握 |
| **DAU/WAU/MAU推移** | 日間/週間/月間アクティブユーザーの推移グラフ。DAU/MAU比のトレンド |
| **リテンション率** | 登録後1日/7日/30日のリテンション率（何%のユーザーが戻ってきているか） |
| **セッション時間** | 平均セッション時間、1日あたりのセッション数 |
| **休眠ユーザー** | 30日以上未ログインユーザーの一覧。休眠率の推移 |

### 10.3 課金・売上分析

| 画面 | 表示内容 |
|------|---------|
| **売上推移グラフ** | 日別/月別の売上推移。折れ線+棒グラフ |

### 10.4 マッチング・バンド分析

| 画面 | 表示内容 |

## 11. アーティスト・曲データの仕組み

アーティストや曲のデータを自前のDBに保持せず、**Spotify API / MusicBrainz から取得**します。DBにはSpotify IDのみ保存し、表示時にAPIから最新情報をRedisキャッシュ経由で取得します。

> 連携要件（エンドポイント・認証方式・レート制限・障害時影響）は [外部連携要件](./11_外部連携要件.md) を、技術実装（データフロー・キャッシュ戦略）は [バックエンド設計](../03_詳細設計/02_バックエンド設計.md) を参照。

## 12. 安全・信頼機能

ユーザー間トラブルの防止と、健全なコミュニティ維持のための機能です。

| 機能 | Free | Pro | 説明 |
|------|------|-----|------|
| 通報 | ○ | ○ | ユーザー・メッセージ・コピバンを通報可能（カテゴリ：詐欺、勧誘、ハラスメント、なりすまし、スパム、その他） |
| ブロック | ○ | ○ | ブロックしたユーザーはマッチング結果・フレンド候補・コピバンメンバー候補から完全除外 |
| レピュテーションスコア | ○ | ○ | ライブ出演・練習参加実績に基づく信頼度表示（★1〜5段階） |
| チャット安全機能 | - | ○ | 禁止ワードフィルタ、URL送信警告、送信レート制限 |

> エスカレーション制度・通報カテゴリ・禁止行為一覧の詳細は [安全対策・リスク管理](./04_安全対策・リスク管理.md) を参照。
- **キャンセルポリシー**: イベント出演の直前キャンセル・無断キャンセルにレピュテーション減少ペナルティ
- **コミュニケーション設計**: 初回接触はアプリ内チャット（Pro） → 信頼構築後にLINE等へ移行するハイブリッド方式

詳細は [04_安全対策・リスク管理.md](./04_安全対策・リスク管理.md) を参照してください。
