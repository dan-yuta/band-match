# Groove Match - アクセシビリティ方針

---

## 1. 基本方針

| 項目 | 内容 |
|------|------|
| 準拠基準 | **WCAG 2.1 Level AA**（フェーズ2以降で段階的に対応） |
| 対象 | Web版（最優先）、iOS/Android（Capacitor WebView経由で同時対応） |
| MVP段階 | Level A の必須項目に対応。Level AA はフェーズ2以降で段階的対応 |

> WCAG（Web Content Accessibility Guidelines）とは、W3Cが策定したWebコンテンツのアクセシビリティに関する国際基準です。

---

## 2. 対応項目

### 2.1 視覚（Level A - MVP必須）

| 項目 | 対応内容 | 実装方法 |
|------|---------|---------|
| **コントラスト比** | テキストと背景のコントラスト比 **4.5:1 以上**（大文字テキストは3:1） | ダークモードUI設計時にコントラストチェッカーで検証 |
| **色だけに依存しない情報伝達** | エラーは色+アイコン+テキストで表示（赤色だけに頼らない） | バリデーションエラーに `⚠️` アイコン + テキストメッセージを併記 |
| **テキストサイズ変更** | ブラウザのフォントサイズ変更（200%）でレイアウトが崩れない | `rem` 単位を使用。固定 `px` を避ける |
| **画像の代替テキスト** | 全ての意味を持つ画像に `alt` 属性を設定 | プロフィール画像: `alt="[ニックネーム]のプロフィール画像"` |

### 2.2 操作（Level A - MVP必須）

| 項目 | 対応内容 | 実装方法 |
|------|---------|---------|
| **キーボード操作** | 全ての操作がキーボードのみで完結可能 | `Tab` で要素間移動、`Enter` で実行、`Escape` でモーダル閉じ |
| **フォーカス表示** | フォーカス中の要素が視覚的に明確 | Tailwind の `focus:ring-2 focus:ring-primary` でフォーカスリング表示 |
| **タッチターゲット** | タッチ操作可能な要素は **44x44px 以上** | ボタン・リンクの最小サイズを保証 |

### 2.3 理解しやすさ（Level A - MVP必須）

| 項目 | 対応内容 | 実装方法 |
|------|---------|---------|
| **言語設定** | HTMLに `lang="ja"` を設定 | `<html lang="ja">` |
| **エラーの特定** | フォームエラー時に該当フィールドを特定し修正方法を提示 | 「パスワードは8文字以上で英字と数字を含めてください」 |
| **ラベル** | 全フォーム要素に `<label>` を紐づけ | `<label htmlFor="email">メールアドレス</label>` |

### 2.4 フェーズ2以降（Level AA）

| 項目 | 対応内容 | 時期 |
|------|---------|------|
| スクリーンリーダー対応 | ARIA属性（`aria-label`, `aria-describedby`, `role`）の追加 | フェーズ2 |
| 動画・音声コンテンツ | 字幕・テキスト代替（カバー動画機能追加時） | フェーズ3 |
| 読み上げ順序 | DOM順序と視覚的順序の一致を保証 | フェーズ2 |

## 3. ダークモードのアクセシビリティ

Groove Matchはダークモード基調のUIです。ダークモード特有のアクセシビリティ課題に対応します。

| 課題 | 対応 |
|------|------|
| 暗い背景での文字視認性 | テキスト色は `#e2e8f0` 以上の明度を確保（コントラスト比 7:1 以上推奨） |
| グラスモーフィズムの視認性 | 半透明カードの上のテキストは背景に関わらず読めるよう、カード背景の不透明度を十分に確保 |
| ステータス色の区別 | 成功（緑）/警告（黄）/エラー（赤）はアイコン+テキストを併記 |

## 4. 多言語対応方針

### 現時点

| 項目 | 内容 |
|------|------|
| 対応言語 | **日本語のみ** |
| UI言語 | 日本語固定 |
| コンテンツ | 日本語 |

### 将来（海外展開検討時）

| 項目 | 内容 |
|------|------|
| 実装方式 | `next-intl`（Next.js用の国際化ライブラリ） |
| 対応候補言語 | 英語（第1優先）、韓国語、中国語（簡体字） |
| 翻訳方式 | プロフェッショナル翻訳 + 機械翻訳の併用 |
| ロケール戦略 | URLパス方式（`/en/dashboard`, `/ja/dashboard`） |
| 検討トリガー | 海外からのアクセスが月間10%を超えた場合 |
