## 1. SEO（検索エンジン最適化）

### 1.1 基本設定

| 項目 | 実装 |
|------|------|
| `<title>` タグ | ページごとに固有のタイトル。例: 「コピバン仲間を探す - Groove Match」 |
| `<meta name="description">` | ページごとに固有の説明文（120文字以内） |
| OGP（Open Graph Protocol） | `og:title`, `og:description`, `og:image`, `og:url` を全ページに設定 |
| Twitter Card | `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image` |
| canonical URL | `<link rel="canonical">` で正規URLを指定（重複コンテンツ防止） |
| sitemap.xml | 公開ページ（ランディング・イベント一覧等）の自動生成 |
| robots.txt | 管理画面・API・認証ページをクロール除外 |
| 構造化データ | JSON-LD でイベント情報（`Event`スキーマ）を記述 |

### 1.2 ページ別SEO対策

| ページ | タイトル例 | 狙うキーワード |
|--------|----------|--------------|
| ランディング | 「Groove Match - コピバン仲間を見つけてライブに出よう」 | コピバン メンバー募集, バンドメンバー探し |
| イベント一覧 | 「コピバンライブイベント一覧 - Groove Match」 | コピバン ライブ, 初心者 ライブ |
| イベント詳細 | 「[イベント名] - [地域] - Groove Match」 | [地域] コピバン ライブ |

### 1.3 技術的SEO

| 項目 | 対応 |
|------|------|
| SSR / SSG | ランディングページ・イベント一覧はSSR（Server-Side Rendering）で配信しクローラーに対応 |
| Core Web Vitals | LCP < 2.5秒、FID < 100ms、CLS < 0.1 を目標 |
| モバイルフレンドリー | レスポンシブデザイン（モバイルファースト） |
| HTTPS | 全ページHTTPS（SEOランキング要因） |
| ページ速度 | CloudFront CDN + 画像最適化（next/image） |
