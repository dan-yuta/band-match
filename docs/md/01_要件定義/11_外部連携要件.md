## 2. 各連携の要件

### 2.1 Spotify Web API


| 項目              | 要件                                                                    |
| --------------- | --------------------------------------------------------------------- |
| **利用目的**        | ユーザーが「やりたい曲」「弾ける曲」を検索・登録するため                                          |
| **使用するエンドポイント** | 曲検索（`/v1/search`）、曲詳細（`/v1/tracks/{id}`）、アーティスト情報（`/v1/artists/{id}`） |
| **認証方式**        | OAuth 2.0 Client Credentials Flow（サーバーサイド）                            |
| **データの扱い**      | Spotify IDのみDBに保存。曲名・アーティスト名・画像はキャッシュ経由で都度取得                          |
| **キャッシュ**       | Redis（検索結果1時間、曲情報24時間、アクセストークン50分）                                    |
| **レート制限**       | 25〜50 req/秒（Tier別）                                                    |
| **商用利用審査**      | 必要。リジェクト時はMusicBrainzにフォールバック                                         |
| **障害時の影響**      | 曲検索・曲情報表示が不可。マッチングスコア計算はSpotify IDの比較のみなので影響なし                        |


### 2.2 Stripe


| 項目              | 要件                                                                                    |
| --------------- | ------------------------------------------------------------------------------------- |
| **利用目的**        | Web版のProプラン月額決済（¥500/月）                                                               |
| **使用する機能**      | Stripe Checkout（リダイレクト方式）、Webhook、Subscription管理                                      |
| **PCI DSS**     | カード情報をサーバーに一切保持しない（SAQ-A準拠）                                                           |
| **Webhookイベント** | `checkout.session.completed`、`customer.subscription.deleted`、`invoice.payment_failed` |
| **Webhook署名検証** | 必須（HMAC-SHA256）                                                                       |
| **障害時の影響**      | 新規決済不可。既存サブスクの利用は継続可能                                                                 |


### 2.3 AWS Cognito


| 項目           | 要件                                       |
| ------------ | ---------------------------------------- |
| **利用目的**     | ユーザー認証、ソーシャルログイン統合                       |
| **対応プロバイダー** | メール+パスワード、Google、X (Twitter)、Instagram   |
| **トークン管理**   | アクセストークン（24時間）、リフレッシュトークン（30日）           |
| **無料枠**      | 1万MAUまで無料                                |
| **障害時の影響**   | ログイン・新規登録が不可。既にログイン中のユーザーはトークン有効期限まで利用可能 |


### 2.4 OpenAI Embedding API


| 項目          | 要件                                         |
| ----------- | ------------------------------------------ |
| **利用目的**    | ユーザープロフィールのテキストをベクトル化し、マッチング精度を向上          |
| **使用モデル**   | `text-embedding-3-small`                   |
| **実行タイミング** | プロフィール更新時に非同期でEmbedding生成                  |
| **保存先**     | RDS PostgreSQL（pgvector拡張）                 |
| **対象データ**   | 自己紹介文・ジャンルテキスト・アーティスト名（**匿名化して送信**）        |
| **障害時の影響**  | Embedding更新が遅延するのみ。既存のEmbeddingでマッチングは継続可能 |


### 2.5 RESAS API


| 項目          | 要件                                     |
| ----------- | -------------------------------------- |
| **利用目的**    | 都道府県・市区町村マスタデータの取得                     |
| **キャッシュ**   | Redis（TTL: 30日）                        |
| **フォールバック** | 静的JSONファイルを同梱。API障害時はキャッシュ→静的JSONの順で提供 |
| **障害時の影響**  | キャッシュがある限り影響なし                         |

## 3. 障害時の影響まとめ


| 外部サービス                  | 障害時のユーザー影響               | 対策                              |
| ----------------------- | ------------------------ | ------------------------------- |
| Spotify API             | 曲検索不可。既存データの表示はキャッシュから継続 | MusicBrainzフォールバック + Redisキャッシュ |
| Stripe                  | 新規決済不可。既存サブスクは影響なし       | status.stripe.com監視。手動リトライ      |
| Cognito                 | ログイン・新規登録不可              | トークン有効期限内のユーザーは利用継続可能           |
| OpenAI                  | Embedding更新遅延            | 既存Embeddingでマッチング継続。バッチリトライ     |
| RESAS                   | 影響なし（キャッシュ+静的JSON）       | 30日キャッシュ + 静的JSONフォールバック        |
| App Store / Google Play | モバイル課金不可                 | Web版Stripe決済に誘導                 |
