# Groove Match - プライバシー・データ管理方針

> 本ドキュメントは、Groove Matchが収集・保存・利用するユーザーデータの種類、保持期間、削除方針、第三者提供の範囲を定義します。
> プライバシーポリシー（利用規約に掲載する法的文書）の元となる技術仕様書です。

---

## 1. 収集するデータの分類

### 1.1 ユーザーが直接入力するデータ

| データ項目 | 必須/任意 | 保存先 | 暗号化 | 用途 |
|-----------|----------|--------|:------:|------|
| メールアドレス | 必須 | Cognito + RDS | ○ | 認証、通知送信 |
| パスワード | 必須 | Cognito | ○（bcrypt） | 認証 |
| ニックネーム | 必須 | RDS | ─ | 表示名 |
| 生年月日 | 必須 | RDS | ○ | 年代表示の計算用（「20代前半」等。生年月日そのものは非公開） |
| 都道府県・市区町村 | 必須 | RDS | ─ | マッチングの地域スコア |
| 楽器・ジャンル・スキルレベル | 必須 | RDS | ─ | マッチングスコア計算 |
| やりたい曲・アーティスト | 必須 | RDS（Spotify IDのみ） | ─ | マッチング、セットリスト |
| 活動可能スケジュール | 必須 | RDS | ─ | マッチングスコア計算 |
| 自己紹介文 | 任意 | RDS | ─ | プロフィール表示、AIマッチング |
| プロフィール画像 | 任意 | S3 | ○（SSE-S3） | プロフィール表示 |
| SNSアカウント（X/Instagram/LINE/YouTube） | 任意 | RDS | ─ | プロフィール表示（フレンドのみ公開を検討） |

### 1.2 自動収集するデータ

| データ項目 | 保存先 | 保持期間 | 用途 |
|-----------|--------|---------|------|
| ログイン日時・最終アクティブ日時 | RDS | アカウント存続中 | オンライン表示、アクティブユーザー判定 |
| IPアドレス | CloudWatch Logs | 30日 | 不正アクセス検出、セキュリティ調査 |
| ユーザーエージェント（ブラウザ/OS情報） | CloudWatch Logs | 30日 | デバイス別最適化、不正検出 |
| マッチング行動（いいね/スキップ） | RDS | アカウント存続中 | AIマッチング精度向上、KPI計測 |
| 練習ログ（曲・時間・日付） | RDS | アカウント存続中 | ストリーク・バッジ・ランキング |
| チャットメッセージ | RDS | アカウント存続中 | チャット表示、通報時の証拠確認 |
| プッシュ通知トークン | RDS | アカウント存続中 | プッシュ通知配信 |
| Embedding ベクトル | RDS（pgvector） | プロフィール更新時に再生成 | AIマッチング |

### 1.3 収集しないデータ

| データ項目 | 理由 |
|-----------|------|
| **クレジットカード番号** | Stripe/Apple/Googleに完全委託。サーバーに一切保持しない |
| **位置情報（GPS）** | 収集しない。地域は都道府県・市区町村の選択のみ |
| **連絡先（電話帳）** | 収集しない |
| **通話・音声データ** | 収集しない（音声通話機能なし） |
| **実名（本名）** | 収集しない。ニックネームのみ |

## 2. データの保持期間と削除

### 2.1 アクティブユーザー

アカウントが有効な間は全データを保持する。

### 2.2 アカウント削除時

ユーザーがアカウント削除を要求した場合（F-07）、以下のスケジュールでデータを処理する。

| データ | 処理 | タイミング |
|--------|------|-----------|
| プロフィール情報（名前、自己紹介、楽器、ジャンル等） | **完全削除** | 即時 |
| メールアドレス | **ハッシュ化して保持**（再登録防止用） | 即時ハッシュ化、ハッシュは1年後に削除 |
| プロフィール画像 | **S3から完全削除** | 即時 |
| チャットメッセージ | **送信者名を「退会済みユーザー」に置換**。メッセージ本文は相手側の記録として保持 | 即時 |
| 練習ログ・ストリーク | **完全削除** | 即時 |
| マッチング履歴 | **完全削除** | 即時 |
| バンドメンバーシップ | **脱退処理**。バンド自体は残る | 即時 |
| 通報履歴（被通報） | **匿名化して保持**（悪質ユーザーの再登録対策） | 匿名化は即時、1年後に完全削除 |
| Stripe顧客ID | **Stripeの顧客データ削除APIを呼び出し** | 即時 |
| Cognito ユーザー | **Cognito User Poolから削除** | 即時 |
| CloudWatch ログ | **通常のログ保持期間（30日）で自動削除** | 自動 |
| Embedding ベクトル | **完全削除** | 即時 |

### 2.3 休眠アカウント

| 条件 | アクション |
|------|-----------|
| 6ヶ月間ログインなし | メールで「アカウントが休眠状態です」と通知 |
| 12ヶ月間ログインなし | 再度メールで通知 + アプリ内表示を「休眠中」に変更。マッチング結果から除外 |
| 24ヶ月間ログインなし | 最終通知メール送信。30日以内にログインがなければアカウントデータを削除 |

> 有料サブスクリプションがアクティブな場合は、休眠アカウント処理の対象外とする。

## 3. 第三者への提供

### 3.1 提供する場合

| 提供先 | 提供するデータ | 目的 | 法的根拠 |
|--------|-------------|------|---------|
| **Stripe** | メールアドレス、サブスクリプション状態 | 決済処理 | 契約の履行 |
| **Apple / Google** | 購入トークン | アプリ内課金の検証 | 契約の履行 |
| **Spotify API** | 検索クエリ（曲名・アーティスト名） | 曲データ取得 | 正当な利益 |
| **OpenAI** | 自己紹介文・ジャンルテキスト（匿名化） | Embedding生成 | 同意 |
| **AWS** | 全データ（インフラとして） | サービス提供基盤 | 契約の履行 |

### 3.2 提供しない場合

| 提供しない相手 | 理由 |
|--------------|------|
| 広告事業者 | 広告モデルではないため不要 |
| データブローカー | ユーザーデータの販売は行わない |
| 他のユーザー（プロフィール公開範囲外） | ブロック・プライバシー設定を尊重 |

### 3.3 法執行機関への開示

裁判所の令状、または法令に基づく正式な要請があった場合に限り、必要最小限のデータを開示する。ユーザーには事前通知する（法令で禁じられている場合を除く）。

## 4. データの暗号化

| レイヤー | 要件 |
|---------|------|
| 通信時（In Transit） | 全通信をHTTPS/TLS で暗号化。HTTP通信は許可しない |
| 保存時（At Rest） | DB・ストレージ・バックアップを暗号化 |
| パスワード | 不可逆なハッシュ化で保存（平文保存禁止） |
| 生年月日 | アプリケーションレベルで暗号化（年代計算以外でのアクセス防止） |

> 暗号化の具体的な方式（TLS 1.2、AES-256、bcrypt等）は [バックエンド設計](../03_詳細設計/02_バックエンド設計.md) および [非機能要件](./10_非機能要件.md) を参照。

## 5. プライバシーポリシーに記載すべき事項

利用規約・プライバシーポリシー作成時に、以下の事項を必ず含める。

| 項目 | 内容 |
|------|------|
| 収集するデータの種類 | 本文書のセクション1に基づく |
| 利用目的 | マッチング、コミュニケーション、サービス改善 |
| 第三者提供 | Stripe・Apple・Google・OpenAI・AWS |
| データの保持期間 | 本文書のセクション2に基づく |
| ユーザーの権利 | アクセス権、訂正権、削除権、データポータビリティ権 |
| Cookieの使用 | セッション管理、認証トークンの保持 |
| 18歳未満の利用 | 利用不可（利用規約で禁止）。初回登録時のクレジットカード/決済方法登録が年齢フィルタとして一定の効果を持つ。将来的に身分証確認の導入を検討 |
| プライバシーポリシーの変更 | 変更時にメール通知 + アプリ内通知 |
| チャットメッセージの自動スキャン | 禁止ワード検出・個人情報検出等の安全機能のため、チャットメッセージを自動スキャンする旨を告知 |
| 問い合わせ先 | プライバシーに関する窓口 |

## 6. GDPR・個人情報保護法への対応

現時点では日本国内向けサービスだが、将来の海外展開に備えてGDPR（EU一般データ保護規則）の基本原則にも配慮する。

| GDPR原則 | Groove Matchの対応 |
|---------|------------------|
| 合法性・公正性・透明性 | 利用規約・プライバシーポリシーで明示。クリックラップ方式で同意取得 |
| 目的の制限 | マッチング・コミュニケーション・サービス改善以外にデータを使わない |
| データの最小化 | 必要最小限のデータのみ収集（実名・GPS・電話帳は収集しない） |
| 正確性 | ユーザーがプロフィールを随時編集可能 |
| 保存の制限 | 休眠アカウントの自動削除（24ヶ月） |
| 完全性・機密性 | 暗号化（AES-256、TLS 1.2）、アクセス制御 |
| アカウンタビリティ | データ処理の記録、プライバシー影響評価の実施（将来） |

### データ主体の権利

| 権利 | 実装方法 |
|------|---------|
| **アクセス権** | プロフィール画面で自分のデータを確認可能 |
| **訂正権** | プロフィール編集機能 |
| **削除権（忘れられる権利）** | アカウント削除機能（F-07）。セクション2.2の手順で処理 |
| **データポータビリティ権** | 設定画面から「自分のデータをエクスポート」機能を提供（JSON形式） |
| **処理の制限権** | アカウント一時停止機能 |
| **異議申立権** | AIマッチングのオプトアウト設定（ルールベースのみに切替可能） |
