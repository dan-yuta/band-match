# Groove Match - 通知設計

> 本ドキュメントは、Groove Matchの通知システム（プッシュ通知・メール通知・アプリ内通知）の設計を定義します。

---

## 1. 通知チャネル

| チャネル | 技術 | 用途 | Pro/Free |
|---------|------|------|---------|
| **プッシュ通知** | APNs（iOS）/ FCM（Android）/ Web Push | リアルタイム性の高い通知 | Pro: 全種類、Free: 基本のみ |
| **メール通知** | Amazon SES | 重要な通知、トライアル終了、アカウント関連 | 共通 |
| **アプリ内通知** | RDS + WebSocket | アプリ起動中に確認する通知 | 共通 |

---

## 2. 通知一覧

### 2.1 マッチング関連

| 通知 | チャネル | トリガー | 対象 | 優先度 |
|------|---------|---------|------|--------|
| マッチング成立 | プッシュ + アプリ内 | 相互いいね成立時 | 両ユーザー | 高 |
| いいね受信 | アプリ内のみ | いいねを受信した時 | 受信者（Pro） | 中 |
| 新しいおすすめ | プッシュ + アプリ内 | 週1回（月曜朝9時） | 全ユーザー | 低 |

### 2.2 チャット関連（Pro限定）

| 通知 | チャネル | トリガー | 対象 | 優先度 |
|------|---------|---------|------|--------|
| 新着メッセージ | プッシュ + アプリ内 | メッセージ受信時（チャット画面を開いていない場合のみ） | 受信者 | 高 |
| 練習日程提案 | プッシュ + アプリ内 | バンドメンバーが練習日程を提案した時 | バンドメンバー全員 | 高 |
| 練習日程確定 | プッシュ + アプリ内 | リーダーが練習日程を確定した時 | バンドメンバー全員 | 高 |
| 練習リマインド | プッシュ | 確定済み練習日程の前日18時 | 参加予定メンバー | 高 |

### 2.3 バンド関連

| 通知 | チャネル | トリガー | 対象 | 優先度 |
|------|---------|---------|------|--------|
| バンド招待 | プッシュ + アプリ内 | メンバー招待を受けた時 | 招待された人 | 高 |
| メンバー参加 | アプリ内 | 新メンバーがバンドに参加した時 | バンドメンバー | 中 |
| メンバー脱退 | アプリ内 | メンバーがバンドを脱退した時 | バンドリーダー | 中 |

### 2.4 イベント関連

| 通知 | チャネル | トリガー | 対象 | 優先度 |
|------|---------|---------|------|--------|
| 地域のイベント | プッシュ + アプリ内 | 自分の活動地域に新しいイベントが登録された時 | 該当地域ユーザー | 中 |
| イベントリマインド | プッシュ | 応募済みイベントの3日前 | 応募者 | 高 |
| 出演バンド確定 | プッシュ + アプリ内 | 自分のバンドがイベント出演確定した時 | バンドメンバー | 高 |

### 2.5 練習トラッカー関連

| 通知 | チャネル | トリガー | 対象 | 優先度 |
|------|---------|---------|------|--------|
| ストリーク危機 | プッシュ | 当日21時時点で練習記録がない場合（ストリーク2日以上の場合のみ） | 本人 | 中 |
| バッジ獲得 | アプリ内 | バッジ獲得条件を満たした時 | 本人 | 低 |
| フレンド練習報告 | アプリ内 | フレンドが練習を記録した時（日次サマリー、個別通知はしない） | フレンド | 低 |
| ランキング変動 | アプリ内 | 週間ランキングで順位が変動した時（月曜朝） | ランキング対象者 | 低 |

### 2.6 アカウント・課金関連

| 通知 | チャネル | トリガー | 対象 | 優先度 |
|------|---------|---------|------|--------|
| アカウント登録完了 | メール | 登録完了時 | 本人 | 必須 |
| メール認証 | メール | 登録時 | 本人 | 必須 |
| パスワードリセット | メール | リセット要求時 | 本人 | 必須 |
| トライアル終了3日前 | メール + プッシュ | トライアル終了3日前 | トライアル中ユーザー | 必須 |
| トライアル終了（課金開始） | メール | トライアル終了日 | ユーザー | 必須 |
| 決済失敗 | メール + プッシュ | Stripe Webhookで決済失敗を検知 | 本人 | 必須 |
| プラン変更完了 | メール + アプリ内 | プラン変更時 | 本人 | 高 |
| 休眠アカウント警告 | メール | 6ヶ月/12ヶ月/24ヶ月未ログイン | 本人 | 中 |

| 通知 | チャネル | トリガー | 対象 | 優先度 |
|------|---------|---------|------|--------|
| 通報受理 | メール + アプリ内 | 通報が処理された時 | 通報者 | 高 |

| チェックイン確認 | プッシュ | 練習開始から2時間後（オプトイン） | 本人 | 中 |

## 3. 通知頻度の制限

ユーザーが通知過多にならないよう、以下の制限を設ける。

| 制限 | ルール |
|------|--------|
| プッシュ通知の1日上限 | **最大10件/日**（アカウント・課金関連は除外） |
| 同一種別の再送間隔 | **最低1時間**（例: 「ストリーク危機」は1日1回のみ） |
| 夜間制限 | 22時〜8時はプッシュ通知を送信しない（アカウント・セキュリティ関連は除外） |
| バッチ送信 | フレンド活動は個別通知せず日次サマリーにまとめる |

## 4. 通知のカスタマイズ（Pro限定）

Pro会員は以下の通知をON/OFF切り替え可能。

| カテゴリ | デフォルト | カスタマイズ可否 |
|---------|:---------:|:--------------:|
| マッチング成立 | ON | ○ |
| チャット新着 | ON | ○ |
| 練習リマインド | ON | ○ |
| ストリーク危機 | ON | ○ |
| イベント情報 | ON | ○ |
| フレンド活動 | ON | ○ |
| ランキング変動 | OFF | ○ |
| アカウント・課金 | ON | **×（変更不可）** |
| 安全・警告 | ON | **×（変更不可）** |

Free会員はデフォルト設定のみ。カスタマイズ機能はPro特典。

## 5. 技術実装

### プッシュ通知のアーキテクチャ

```
通知トリガー（API/WebSocket/バッチ）
    ↓
Lambda（通知ハンドラー）
    ↓
Amazon SNS（プッシュ通知プラットフォーム）
    ├── APNs（iOS）
    ├── FCM（Android）
    └── Web Push（ブラウザ）
```

### メール通知のアーキテクチャ

```
通知トリガー
    ↓
SQS（メールキュー）
    ↓
Lambda（メール送信ワーカー）
    ↓
Amazon SES（メール送信）
    ↓
ユーザーのメールボックス
```

> SQSを挟むことで、大量メール送信時のレート制限とリトライを管理する。

### アプリ内通知

```
通知トリガー
    ↓
notifications テーブルに INSERT
    ↓
WebSocket（リアルタイム）で接続中ユーザーに即時配信
    ↓
未接続ユーザーは次回アプリ起動時にAPI（GET /api/notifications）で取得
```

| テーブル: notifications | 型 | 説明 |
|------------------------|-----|------|
| id | UUID | 通知ID |
| user_id | UUID | 通知先ユーザー |
| type | ENUM | 通知種別 |
| title | VARCHAR | タイトル |
| message | TEXT | 本文 |
| link | VARCHAR | タップ時の遷移先 |
| read | BOOLEAN | 既読フラグ |
| created_at | TIMESTAMP | 作成日時 |
