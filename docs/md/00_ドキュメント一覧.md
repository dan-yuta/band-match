### まず全体像を把握する
1. **01_アプリ概要** → Groove Matchのビジョン・ターゲット・ペルソナ・KPI
2. **02_機能一覧** → 全機能と課金モデルの把握
3. **01_マッチングアルゴリズム** → コア機能のロジックを理解

4. **01_フロントエンド** → 画面構成・コンポーネント・ルーティング
5. **02_バックエンド設計** → API・DB・認証・外部API連携・決済セキュリティ
6. **02_インフラ構成** → AWS構成と3環境の概要・非機能要件

### ビジネス・運用を把握する
7. **01_コスト計算** → 3環境のコストと損益分岐点
8. **02_運用・保守** → メンテナンス・バックアップ・監視
9. **03_競合分析** → 競合との差別化ポイント

### 安全対策・制約を理解する
10. **04_安全対策・リスク管理** → トラブル予防・通報・コミュニケーション設計・免責方針
11. **05_制約・前提条件** → 開発体制・予算・法的制約・ローンチ戦略

### 開発に参加する場合
12. **01_起動・開発手順** → ローカル環境構築・テストアカウント・トラブルシューティング
13. **00_用語集** → 不明な用語はここを参照
