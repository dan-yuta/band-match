Spotify API データフロー

曲データの検索・保存・表示フロー（Spotify IDのみDB保存）

🔍 曲を登録するとき

① ユーザーが曲名を入力
↓
② Spotify API で検索（リアルタイムサジェスト）
↓
③ ユーザーが曲を選択
↓
④ Spotify track ID だけをDBに保存
曲名・アーティスト名はDBに保存しない

🎵 曲を表示するとき

① DBから Spotify track ID を取得
↓
② Redis キャッシュを確認
TTL: 曲情報24時間 / 検索結果1時間
↓ キャッシュなし
③ Spotify API から最新情報取得
↓
④ 曲名・アーティスト名・ジャケット画像を表示

セキュリティ多層防御

フロントエンドからデータベースまでの全レイヤーで防御

🌐 CloudFront + WAF
DDoS防御、ボット対策、レートリミット、SQLi/XSSフィルタ
🔒 HTTPS（TLS 1.2以上）
全通信を暗号化。ACM証明書。HTTP通信は一切許可しない
🔐 Cognito認証 + JWT
パスワード: bcrypt暗号化。MFA対応。クロスタブ同期
🛡️ API Routes ミドルウェア
入力バリデーション、CSRF対策、認証ガード、管理者権限チェック
🗄️ Prisma ORM
パラメータバインディングでSQLインジェクション防止
🔑 RDS暗号化（AES-256）+ Secrets Manager
保存データ暗号化。APIキー・DB接続情報を安全に管理。自動ローテーション
