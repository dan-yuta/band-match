アカウント削除時のデータ処理

ユーザーがアカウント削除を要求した場合の処理

✅ 即時完全削除

🗑️プロフィール情報（名前・自己紹介等）
🗑️プロフィール画像（S3から削除）
🗑️練習ログ・ストリーク
🗑️マッチング履歴
🗑️Embeddingベクトル
🗑️Cognito ユーザー
🗑️Stripe顧客データ（API経由で削除）

⏳ 匿名化・遅延削除

🔄メールアドレス → ハッシュ化（1年後に削除）
🔄チャットメッセージ → 送信者名を「退会済みユーザー」に置換
🔄通報履歴（被通報）→ 匿名化保持（1年後削除）
🔄バンドメンバーシップ → 脱退処理（バンドは残る）
🔄CloudWatch ログ → 30日で自動削除

休眠アカウント処理

長期間未ログインのアカウントは段階的に処理

6ヶ月
メール通知「アカウントが休眠状態です」
12ヶ月
再通知 + マッチング結果から除外「休眠中」表示に変更
24ヶ月
最終通知 → 30日以内にログインがなければアカウントデータ削除
⚠️ 有料サブスクリプションがアクティブな場合は休眠処理の対象外

暗号化レイヤー

🌐 通信時（In Transit）: TLS 1.2以上 — HTTPS必須
🗄️ 保存時（At Rest）: AES-256 — RDS, S3, EBS, バックアップ
🔑 パスワード: bcrypt — Cognito標準
🎂 生年月日: アプリケーションレベル暗号化 — 年代計算以外でのアクセス防止
