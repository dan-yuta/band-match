📊 現行アルゴリズム評価

現行スコアリングは6要素の重み付き合計（最大160点 → 100点にクランプ）で構成される。 シンプルで高速・低コストだが、深い好みの理解には限界がある。

低コスト・ゼロAPIコスト 外部API不要。DBクエリのみで完結し、運用コストが極めて低い。
🔒
決定論的・再現性 同じ入力に対して常に同じ結果。デバッグ・QAが容易。
📖

超低レイテンシ P50: 5ms / P95: 20ms。リアルタイムUI更新も問題なし。
🎸

🖥️
追加インフラ不要 既存RDS上で動作。pgvectorもSageMakerも不要。
⚠️弱み（5項目）
🧠
深い好みの理解不足 「明るい人が好み」「初心者に優しいバンド」等の意味的マッチング不可。
🆕
コールドスタート問題 新規ユーザーはデータが少なく、低スコアになりやすい。
📚
学習・改善機能なし ユーザーの行動（いいね・スキップ）からの自動改善が不可能。
⚖️
重みが固定（エンジニア依存） 地域40 vs 曲35の最適バランスは実データで検証が必要。手動調整のみ。
🎵
曲未登録 = スコアゼロ問題 DBに曲が登録されていない場合、曲スコアが0になり不公平なマッチング。

🚀 なぜ最初からAIか？

「まず現行スコアリングで動かして、後でAIに移行すればいい」という考えが 実はコスト高・リスク大になる理由を解説する。

😰
後から移行するコストは予想以上に高い
後からAIを組み込む場合、単純な「機能追加」では済まない。 DBスキーマの変更（embeddingカラム追加・インデックス再構築）、 マッチングロジックの全面書き換え、A/Bテスト基盤の構築、 既存データの埋め込みバッチ生成など、工数が膨大になる。
推定移行工数: 5〜10営業日 × エンジニア人件費 = ¥50万〜100万
📊
ユーザーデータの早期蓄積効果
AIの精度は蓄積されたユーザー行動データと正比例する。 ローンチからEmbeddingを有効化することで、ユーザーのプロフィール・ いいね・スキップなどの行動データが最初から最適化され続ける。 6ヶ月の遅れが協調フィルタリング導入を半年遅らせることになる。
早期蓄積効果: MAU 1K × 6ヶ月 = 6,000件の学習データ先行取得
💾
pgvectorのコストは驚くほど安い
pgvectorはRDSの拡張機能として利用可能。 既存RDSインスタンスを一段アップグレードするだけで有効化できる。 専用ベクターDBやSageMakerは不要。 MAU 1K規模では月¥3,000〜5,000の追加コストで済む。
RDS アップグレード（db.t3.medium → db.t3.large）: +¥3,000〜5,000/月
🔑
OpenAI Embedding APIは実質無料に近い
text-embedding-3-small は $0.02/1M tokens と極めて安価。 プロフィール更新時のみEmbeddingを再生成し、ベクターをキャッシュすることで、 マッチング検索自体はpgvectorのローカル計算のみ（API追加コストゼロ）。 MAU 1Kで月¥3、MAU 20Kでも月¥60以下。
MAU 1K: ¥3/月 → MAU 5K: ¥15/月 → MAU 20K: ¥60/月（キャッシュ込み）

🔄 「後から移行」vs「最初から組み込む」の比較

😰 後からAIに移行する場合
  • ❌DBスキーマ変更（embedding列追加、インデックス再構築、マイグレーション）
  • ❌マッチングロジック全面書き換え（テスト含む）
  • ❌A/Bテスト基盤の後付け構築（既存ユーザーへの影響考慮が複雑）
  • ❌既存全ユーザーのEmbeddingバッチ生成（移行コスト一括発生）
  • ❌ユーザー体験の変化によるクレーム・サポート対応
  • ❌6〜12ヶ月分の学習データが欠損（精度向上が遅れる）
  • ❌エンジニア工数 5〜10営業日（¥50〜100万相当）
💸 トータル追加コスト: ¥50万〜100万 + 機会損失
🚀 最初から組み込む場合
  • ✅初期設計にpgvectorを含める（追加工数1〜2日のみ）
  • ✅ハイブリッドロジックをローンチから実装（設計が一貫）
  • ✅A/Bテスト基盤を最初から設計（将来の実験が容易）
  • ✅新規ユーザー登録時に自動でEmbedding生成（バッチ不要）
  • ✅ユーザーは最初からAI精度のマッチングを体験
  • ✅ローンチ初日から学習データ蓄積開始
  • ✅追加コスト: pgvector +¥5K/月 + Embedding +¥3〜60/月
💎 トータル追加コスト: ¥5,063〜/月（初月から収益化可能）
🎯
結論：ハイブリッドAIをDay 1から採用すべき
追加コスト¥5,000〜/月（pgvector + Embedding）は、後から移行する場合の¥50万〜100万と比較すると 100倍以上のコスト効率がある。 さらに、早期からデータを蓄積することで協調フィルタリングなど高度なAIへの移行も スムーズに行える。ユーザー体験の一貫性を保ちつつ、最初から競合優位性を確立できる。 Groove Matchのポジショニング「AI駆動のバンドマッチング」はDay 1から実現すべきである。
